# 快速开始指南

## 🚀 启动服务

```bash
./serve.sh
```

然后在浏览器打开：**http://localhost:40000**

## 📋 常用命令

```bash
./serve.sh start      # 启动服务（默认）
./serve.sh stop       # 停止服务
./serve.sh restart    # 重启服务
./serve.sh status     # 查看状态
./serve.sh help       # 查看帮助
```

## 📋 基本使用流程

### 0️⃣ 搜索与筛选
- **搜索**：在顶部搜索框输入关键词，按回车搜索
  - 搜索公司名：显示该公司的所有信息
  - 搜索产品名：仅显示匹配的产品
- **筛选**：点击筛选按钮（互斥选择）
  - "未投递"：只显示未投递的产品
  - "有面试"：只显示有面试记录的产品
  - 同一时间只能激活一个筛选器
- **清除**：点击"清除筛选"恢复全部显示

### 0️⃣ 查看即将到来的面试
- 页面顶部会自动显示今天及以后的所有面试
- 按时间升序排列，最近的在最前
- 橙色醒目提醒样式

### 1️⃣ 添加公司
- 点击右上角"+ 添加公司"
- 输入公司名称（如：字节跳动）
- 勾选是否已创建简历
- 创建后可点击状态标签切换（未创建 → 已创建 → 无需 → 未创建）
  - ✗ 未创建简历（灰色）
  - ✓ 已创建简历（绿色渐变）
  - ○ 无需简历（黄色渐变）

### 2️⃣ 添加产品
- 在公司卡片中点击"+ 产品"
- 输入产品名称（如：抖音推荐、飞书）
- 勾选是否已投递
- 创建后可点击状态标签切换

### 3️⃣ 记录面试
- 在产品下点击"+ 面试"
- 选择面试类型（电话/技术/+1/+2/交叉/HR）
- 选择面试时间
- 可选：添加备注（面试官、题目等）

### 4️⃣ 排序管理
- **拖拽排序**：按住公司/产品左侧的 ⋮⋮ 图标拖动
- **快速置顶/置底**：点击公司卡片的 ⬆️置顶 或 ⬇️置底 按钮
- 排序后自动保存，刷新页面保持顺序

### 5️⃣ 查看统计
系统会自动显示：
- ✅ 具体日期时间（含星期几）
- ✅ 相对时间（3天前）
- ✅ **工作日天数**（自动排除周末）

## 💡 使用技巧

### 状态切换
- **公司状态**：点击"✓ 已创建简历"或"✗ 未创建简历"快速切换
- **产品状态**：点击"✓ 已投递"或"✗ 未投递"快速切换
- 状态改变立即保存，无需额外操作

### 排序技巧
- **拖拽排序**：最灵活，可以精确放到任意位置
- **置顶**：快速将重点公司移到最前面
- **置底**：将不重要的公司移到最后
- 操作后页面滚动位置保持不变，不会跳转

### 工作日计算
- 自动排除周六、周日
- 从面试日期的第二天开始计算
- 显示"距离最后一次面试：X 个工作日"

### 面试排序
- 所有面试记录按时间自动排序
- 轮次编号自动生成（第1轮、第2轮...）

### 数据备份
数据保存在：`backend/data/interv.json`

备份方式：
```bash
# 复制数据文件
cp backend/data/interv.json ~/backup/interv-$(date +%Y%m%d).json

# 或者用 Git 管理
git add backend/data/interv.json
git commit -m "更新面试记录"
```

### 查看日志
```bash
# 后端日志
tail -f /tmp/interv-backend.log

# 前端日志
tail -f /tmp/interv-frontend.log
```

## ⚠️ 常见问题

### Q: 启动失败怎么办？
A: 检查端口是否被占用
```bash
# 检查端口
lsof -i:40000
lsof -i:40001

# 如果被占用，kill 掉
kill -9 <PID>
```

### Q: 前端连接后端失败？
A: 确保后端已启动，检查 `http://localhost:40001/api/companies` 是否可访问

### Q: 数据丢失了？
A: 数据在 `backend/data/interv.json`，建议定期备份

### Q: 如何修改端口？
A: 编辑以下文件：
- 后端：`backend/server.js` 中的 `PORT`
- 前端：`frontend/vite.config.js` 中的 `server.port`
- 前端 API：`frontend/App.vue` 中的 `API_BASE`
